# JSON Editor 仕様メモ（MVP）

## 1. 目的
- 一定期間、少人数の身内利用向けに JSON 編集ツールを作る。
- UI はシンプルに `プルダウン` と `textarea` を中心にする。
- 将来的に Redis へ切り替える前提で、初期実装はモックデータで進める。

## 2. 運用前提
- GitHub への push と Vercel への登録/公開はユーザー側で実施する。
- 本アプリ側は「ローカル実装」「仕様整理」「動作確認」までを対象とする。
- 本番想定は小規模・短期利用のため、実装は簡素さを優先する。

## 3. スコープ
### 対象
- 画面描画時に全件取得し、`title` 一覧をプルダウンへ表示する。
- プルダウン選択時に JSON 内容を textarea に表示する。
- 編集内容を保存できる。
- 「保存してダウンロード」ボタンを押すと保存後に `{タイトル}.json` でダウンロードする。

### 対象外（MVPではやらない）
- 認証/認可
- リアルタイム同時編集制御
- 監査ログ、履歴管理
- 複雑な検索/フィルタ

## 4. 技術スタック（予定）
- Next.js (App Router) + TypeScript
- Vercel（デプロイ先）
- データ層: 初期はモック実装、後で Redis へ差し替え

## 5. UI 仕様
### レイアウト
- 白基調、中央寄せレイアウト
- 横幅は適度に制限（例: `max-width` を設定）
- 情報量は最小限にして、編集に集中できる見た目にする
- スマホ幅でも崩れないように縮小表示する（複雑なレスポンシブは行わない）

### スマホ表示方針（MVP）
- 画面幅 320px 付近でも横スクロールを発生させない
- ボタンは狭い幅で縦並びに切り替える
- textarea の高さはモバイル時に短くして全体を見やすくする

### 画面要素
- プルダウン（タイトル一覧）
- textarea（JSON編集）
- `保存` ボタン
- `保存してダウンロード` ボタン
- ステータスメッセージ（保存成功/失敗、JSONエラー）

### 画面フロー
1. 初期表示で一覧 API を呼び出す
2. 取得した title 一覧をプルダウンへ表示
3. 項目選択で詳細 API を呼び、JSON を textarea に表示
4. `保存` 押下で更新 API を呼ぶ
5. `保存してダウンロード` 押下で更新 API 成功後、JSON をファイル保存する

## 6. 保存＋ダウンロード仕様
- ボタン名: `保存してダウンロード`
- 処理順:
1. textarea の JSON を検証（`JSON.parse`）
2. 更新 API を実行
3. 成功時に JSON 文字列を Blob 化してダウンロード
- ファイル名:
  - 基本: `{タイトル}.json`
  - タイトルが空の場合: `untitled.json`
  - OS 非許容文字は置換して安全な名前にする
- 保存失敗時はダウンロードしない

## 7. モックデータ方針（初期実装）
- Redis 連携の代わりに、モックの Repository 層を用意する
- API からはモックを呼び出すため、将来 Redis 実装へ差し替えやすい形にする
- 例:
  - `GET /api/docs` 一覧
  - `GET /api/docs/:id` 詳細
  - `PUT /api/docs/:id` 更新

## 8. API 仕様（MVP）
### `GET /api/docs`
- 目的: プルダウン用の一覧取得
- 返却: `id`, `title`, `updatedAt`

### `GET /api/docs/:id`
- 目的: 選択中 1 件の取得
- 返却: `id`, `title`, `body`, `updatedAt`

### `PUT /api/docs/:id`
- 目的: JSON 更新
- 入力: `title`, `body`
- 挙動: JSON 妥当性を確認して保存、`updatedAt` 更新

## 9. エラー方針
- JSON 構文不正: 400
- 対象 ID なし: 404
- サーバー内部エラー: 500
- 画面では短いエラーメッセージのみ表示（身内運用向け）

## 12. 開発環境メモ（2026-04-08 時点）
- Node.js は `v20.20.2` を使用
- `npm` は `10.8.2` を使用
- 初回セットアップ済み。依存インストールも完了済み

### ローカル起動コマンド
1. `npm run dev`
2. ブラウザで `http://localhost:3000` を開く

### 補足
- もし `node` / `npm` が見つからない場合は、ターミナルを再起動して PATH を再読込する

## 13. 実装済みエンドポイント（モック）

## 14. VSCode デバッグ
- `.vscode/launch.json` を作成済み
- VSCode の「実行とデバッグ」から次を選択して開始できる

### `Next.js: Debug Server`
- Node 側（API/サーバー処理）のデバッグ向け
- ブレークポイントを `app/api/**` やサーバー処理に置いて利用

### `Next.js: Debug Full Stack (Chrome)`
- Next.js 起動後に Chrome デバッガーを自動で接続
- 画面操作しながらフロント/サーバー両方を追える

### 補足
- デバッグ開始時は `127.0.0.1:3000` を利用
- すでに 3000 番ポートを使っている場合は、既存プロセス停止後に再実行する
